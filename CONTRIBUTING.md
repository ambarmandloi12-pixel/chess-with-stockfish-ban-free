# Contributing to Chess with Stockfish

Thank you for your interest in contributing to this educational chess analysis project!

## 📌 Important Ethical Guidelines

All contributions must adhere to these principles:

1. **Educational Focus**: All improvements should serve educational purposes
2. **Fair Play**: No features designed to cheat or gain unfair advantages
3. **Terms of Service Compliance**: Contributions must not violate any platform's terms of service
4. **Ethical Chess**: Respect the integrity of the game and chess community

## 🔄 How to Contribute

### 1. Fork and Clone
```bash
git clone https://github.com/ambarmandloi12-pixel/chess-with-stockfish-ban-free.git
cd chess-with-stockfish-ban-free
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Keep code clean and well-documented
- Add comments for complex logic
- Follow PEP 8 style guide

### 4. Test Your Changes
- Ensure the program runs without errors
- Test with various chess positions
- Verify no breaking changes

### 5. Commit and Push
```bash
git add .
git commit -m "Add: Brief description of your feature"
git push origin feature/your-feature-name
```

### 6. Submit a Pull Request
- Describe what your PR adds or fixes
- Explain the educational value
- Reference any related issues

## 🎯 Contribution Ideas

Great areas for contributions:

- **Code Quality**: Refactoring, optimization, bug fixes
- **Documentation**: Improving README, adding tutorials
- **Features**: 
  - Support for different board sizes
  - Automated board scanning improvements
  - Better GUI interface
  - PGN file support
  - Opening book integration
  - Endgame tablebase support
- **Testing**: Unit tests, integration tests
- **Analysis Tools**: New analysis features (for learning)

## ❌ What NOT to Contribute

Please do NOT submit:
- Anti-detection mechanisms
- Account protection/evasion features
- Online cheating tools
- Code designed to bypass platform security
- Features to hide engine usage

These violate the educational purpose and ethical guidelines of this project.

## 📚 Documentation

Please include:
- Docstrings for all functions
- Comments for complex logic
- Usage examples in README updates
- Educational context for new features

## 🐛 Reporting Issues

When reporting bugs:
- Describe the issue clearly
- Include steps to reproduce
- Share error messages or logs
- Specify your environment (OS, Python version, etc.)

Feel free to open an issue to discuss ideas before implementing!
